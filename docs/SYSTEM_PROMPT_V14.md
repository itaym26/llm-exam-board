## System Protocol & Architecture

We are building an independent evaluation system based on the Teacher-Student-Judge pattern. Because we are working with standard LLMs rather than prohibitively expensive enterprise tools, we engineered specific defense mechanisms to ensure high reliability and prevent hallucinations:

1. **Truth Gate**: A strict logic filter within the Judge component. It statically scans the student's code for critical logic (e.g., proper mutex locks in multithreading). If critical logic is missing, it zeroes out the LLM's score, strictly overriding the LLM's text feedback to prevent False Positives.
2. **Micro-Resilience (ResilienceUtils)**: Safely extracts JSON from noisy LLM text using regex to prevent system crashes caused by unpredictable API formatting.
3. **Ensemble Consensus (EnsembleManager)**: Aggregates multiple judge scores for the same student answer, calculates the mean, and isolates "Outlier" judges who deviate too far from the consensus.
4. **Audit Gate**: Ensures tasks generated by the Teacher are solvable before passing them to the Student.
5. **Context Isolation**: Ensures the Student model approaches each task with zero prior memory to prevent data leakage.
6. **Itemized Deductions & Dynamic Rubrics**: Forces the judge to return a structured JSON array of specific point deductions based on a dynamic rubric, rather than free-text feedback.

## File Hierarchy & Modules (V14)

The system must be divided into the following hierarchical files:

1. **interfaces.py**: The foundational data models (`TestCase`, `StudentAnswer`, `RubricItem`, `DeductionItem`, `GradedResponse`, `EnsembleResult`).
2. **resilience.py**: Contains `ResilienceUtils` with `clean_json_response(raw_text)` for Micro-Resilience.
3. **generators.py**: Contains `ConfigManager` (topic breakdown) and `QuestionGenerator` (Teacher - generates TestCases, rubrics, and Golden References).
4. **responders.py**: Contains `StudentResponder` (Student - enforces Context Isolation).
5. **judge_engine.py**: Contains `JudgeEngine` (Judge - applies the Truth Gate and parses Itemized Deductions).
6. **orchestrator.py**: Contains `EvaluationOrchestrator` (manages flow and Audit Gate) and `EnsembleManager` (calculates consensus and identifies outliers).
