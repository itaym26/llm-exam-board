# Run 004: Issues Log

## Issue 1: Gemini's free tier has a hard daily cap of 20 requests/model/project, not just a per-minute limit

**Cause:** Starting at `[gemini-2.5-flash #2]`, every subsequent Gemini call failed with `429 RESOURCE_EXHAUSTED`, but the error body is explicit and different from the transient rate-limit errors seen earlier in this project: `quotaId: GenerateRequestsPerDayPerProjectPerModel-FreeTier`, `quotaValue: '20'`. This is a **daily** quota (resets on a calendar-day boundary, not a rolling one-minute window), and it counts every call made against `gemini-2.5-flash` under this API key's project -- including all of this session's earlier diagnostic/debugging calls (finding the right model, testing thinking-budget behavior, run 002, run 003, and the start of this run), not just calls from this specific script execution. By the time run 004 reached its second Gemini-taught question, the day's 20-request budget for this exact model was already exhausted from everything that came before it today.

**How we solved it:** This is not fixable by retrying, backing off, or any client-side change -- the existing `max_retries=3` / 60s timeout hardening in `gemini_client.py` correctly did *not* retry indefinitely against a quota that won't reset for tens of minutes to hours; it surfaced the error immediately, which is the right behavior. Confirmed via the error body itself (`quotaValue: '20'`, `RequestsPerDay`) that this is a hard daily ceiling, not a bug or misconfiguration on our side.

**How we proceeded:** Did not attempt to bypass this (e.g. by requesting a new key from a different Google Cloud project purely to dodge the quota) without checking with the user first, since silently working around a provider's stated free-tier limit isn't something to do unprompted. Documented the actual limit here so future runs can budget for it explicitly: **gemini-2.5-flash's free tier allows only 20 total generate_content calls per day, across every use in this project** (Teacher generation calls and Judge grading calls both count against the same 20). A run plan that includes Gemini in any role should keep total Gemini calls for the day comfortably under 20, or spread testing across multiple days.
